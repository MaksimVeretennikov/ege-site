# Сайт ЕГЭ

## Структура
- `index.html` — разметка
- `css/main.css` — стили
- `js/app.js` — основная логика интерфейса
- `js/storage.js` — работа с localStorage
- `js/sounds.js` — логика звуков
- `js/character.js` — логика персонажа и геймификации
- `js/data.js` — большая база заданий, не относится к UI
- `supabase.js` — клиент Supabase
- `.env.local` — локальные переменные окружения для Supabase, не трогать и не коммитить
- `package.json` — npm-скрипты и зависимости
- `img/` — изображения

## Текущее состояние проекта
- Проект запускается локально через Vite
- `app.js` подключается как ES module
- Supabase уже подключён
- Есть таблица `profiles` в Supabase
- localStorage уже частично вынесен в `js/storage.js`
- проект ещё не переведён с localStorage на Supabase полностью

## Основные правила
- Не анализируй весь проект без необходимости
- Для UI-задач работай только с релевантными файлами:
  - `index.html`
  - `css/main.css`
  - `js/app.js`
  - `js/storage.js`
  - `js/sounds.js`
  - `js/character.js`
- `js/data.js` не открывай и не меняй, если задача не про базу заданий
- `supabase.js` меняй только если задача реально касается Supabase
- `.env.local` не открывай, не меняй, не выводи значения переменных
- Не делай лишний рефакторинг
- Вноси минимальные изменения
- Не трогай нерелевантные файлы
- Перед изменениями кратко перечисляй, какие файлы будут затронуты

## Темы оформления
Для светлой/темной темы:
- используй `data-theme` на `html` или `body`
- сохраняй тему через `Storage.saveTheme()` / `Storage.getTheme()`
- применяй тему при загрузке страницы
- не переписывай весь CSS без необходимости

## Подход к работе
1. Сначала определи нужные файлы
2. Не открывай остальные без необходимости
3. Сначала дай краткий план
4. Делай точечные изменения маленькими шагами
5. После изменений кратко покажи, что именно изменилось
6. Для крупных изменений сначала делай план, потом отдельным шагом реализацию

## Приоритет ближайших изменений
- Сначала перевести пользователей и профиль с localStorage на Supabase
- Затем перевести классы/привязку к классу на Supabase
- Затем перевести сохранение сессий и прогресса на Supabase
- Пока не трогать `js/data.js`
- Пока не делать большой рефакторинг UI
- localStorage пока оставить только для темы и, при необходимости, мелкого UI-состояния
